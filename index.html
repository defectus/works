50 nepravd, kterým programátoři věří

Download as .zip Download as .tar.gz View on GitHub
  1. Anglické jmenné konvence jsou vzorem pro většinu lidí a nikdo to nedělá jinak.
  2. Všechna jména lze zapsat americkou ASCII.
  3. Unicode konečně vyřešil problém kódování textu.
  4. Pohlaví je neměnné a jedná se o seznam dvou možností.
  5. Zákonné jméno člověka je to, jak chce být tento člověk viděn světem.
  6. Obecně vzato, lidé preferují otevřenost před soukromím.
  7. PSČ a poštovní kódy jsou spíše dobrým způsobem jak zjistit místo kde se někdo nachází, než náhodným řetězcem sloužícím k doručení pošty.
  8. Každý má telefon a každé telefonní číslo náleží přesně jednomu člověku.
  9. Objekty jakékoliv velikosti lze kdykoliv doručit komukoliv domů.
  10. Lidé kašlou na bezpečnost.
  11. Technologický průmysl je meritokracií (společností založenou na zásluhách).
  12. Technologický průmysl je zázračně prostý předsudků zbytku společnosti.
  13. Datum a čas jsou přesné, nikoliv vágní.
  14. Právě jsme našli ten pravý formát pro reprezentaci dat: JSON.
  15. Jména mohou být tříděna podle pohlaví.
  16. Služby single sign-on zjednodušují a ulehčují registraci uživatele.
  17. Lidé mají single sign-on pro poskytovatele single sign-on.
  18. Existuje smysluplné rozdělení mezi HTTP zdroji, které se nazývají API a které servírují HTML.
  19. Webová aplikace je rozdílná a smysluplně jiná věc než webová stránka.
  20. CSS může být “objektové” nebo “funkcionální” spíše než deklarativní jazyk pravidel s průměrně složitým systémem dědičnosti.
  21. Unit testy zachytí všechny problémy, které by jinak zachytily kontroly typů a statické analyzéry.
  22. Psaní unit testů je spíše zábava, než únavná nutnost.
  23. Pokud máme kód 100% pokrytý testy, tak je zcela bez chyb.
  24. Metodologie, která nám byla prodána na drahém kurzu nám zaručí, že budeme psát jasně lepší software.
  25. Pokud přepíšeme pochopitelnou zprávu o chybě ("Třída Udělátko vyhodila nezachycenou výjimku, když je na vstupu UTF-8") do rozevláté user story ("jako vývojář, potřebuji aby program běžel a nevypisoval 500 řádek stack tracu"), tak se nám bude zázračně lépe plánovat práce.
  26. Pokud necháme lidi, kteří nemají žádnou zkušenost s zdrojákem, aby udělali rychlé code review zaměřené na styl kódu a jmenné konvence, tak tím zajistíme, že budou vychytané chyby.
  27. Když necháme členy týmu, kteří se nevyznají v jisté části kódu, přijít s náhodným odhadem založeným na jejich tušeních, tak vyřešíme problém plánování a odhadů.
  28. "Rockové hvězdy" vyřeší všechny naše problémy.
  29. Ta bezva nová věc o které se píše na hacker news, vyřeší všechny naše problémy a může být hned bez problému nasazena do produkce.
  30. Bezpečnost je prostě vrstva, která se přidá do softwaru.
  31. GPS je většinou přijatelně přesné ve většině případů.
  32. Jen mobilní zařízení mají geolokaci.
  33. Cokoliv, na čemž běží Windows, MacOS X a neandroidí Linux nemůže být považováno za mobilní zařízení, i když se jedná o mikroskopické super přenosné zařízení, které s sebou můžeme nosit kdokoliv kamkoliv.
  34. Mobilní zařízení mají pomalé připojení, i když si zrovna sedíte na sedačce u televize.
  35. Synchronizace přes Internet je vždy efektivnější a jednodušší než přímé spojení počítačů.
  36. Distribuované VCS je ještě lepší, pokud máte Github jako jediný bod selhání.
  37. Neexistují technické opravy společenských problémů.
  38. Bitcoin je technická oprava společenského problému.
  39. apt-get install bitcoin-qt řeší problém použitelnosti Bitcoinu (to si nevymýšlím).
  40. Lidé většinou jednají racionálně (v klidu, většina ekonomů tomu také věří, bez ohledu na existenci astrologů, homeopatů, teologů, Sazky a proutkařů, kteří tvrdí, že mluví s mrtvými pejsky).
  41. Lidé často aktualizují svůj software.
  42. Pokud má software mnoho voleb, tak je lepší je schovat složitého menu a problém je vyřešen.
  43. Sociální sítě, které používají programátoři na Západě, zhruba odráží sociální sítě, které používají lidé po celém světě.
  44. Výstup z mého barevného BDD udělátka si někdy prohlédne někdo z netechnického managementu.
  45. Hlavička MIME typ přesně značí typ obsahu.
  46. Mít možnost commitovat kód ve výšce 10km na mořem za použití Git (nebo Mercurialu) je to co bych měl dělat, i když je tu možnost si užít alkohol zdarma a tím udělat cestování letadlem o trochu snesitelnějším.
  47. Sekundy od začátku epochy jsou rozumným formátem data (dokonce existuje obecně uznávaná epocha).
  48. Databáze nebo framework, který pozná data založená na časových zónách řeší problém časových zón.
  49. Dohady ohledně metodologie zajistí lepší software.

  50. Nainstalovat homebrew, abych mohl nainstalovat npm, abych mohl nainstalovat bower, abych mohl nainstalovat Angular (apod.) a tím se vyhnul psaní AJAXu, dává smysl.

  51. Software je hrozný.
